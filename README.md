# rpn-calc

RPN-калькулятор на Idris2 с переменными. Пишешь числа и операции через пробел/Enter,
стек растёт и уменьшается, результат виден сразу. Как dc или Forth, только с зависимыми типами.

## Запуск

```bash
pack run rpn-calc
```

## Пример сессии

```
rpn> 3
  стек: 3  <-
rpn> 4
  стек: 3  4  <-
rpn> -
  стек: -1  <-
rpn> dup
  стек: -1  -1  <-
rpn> *
  стек: 1  <-
rpn> let x
  стек: 1  <-
rpn> $x
  стек: 1  1  <-
rpn> +
  стек: 2  <-
```

3 4 - это 3 минус 4, то есть -1. Стандартная RPN-семантика.

## Команды

```
<число>      положить на стек
+ - * /      арифметика (нужно 2 элемента)
neg          сменить знак верхнего
dup          дублировать верхний
drop         удалить верхний
swap         поменять местами два верхних
let имя      сохранить верхний в переменную
$имя         загрузить переменную на стек
dump         показать стек
vars         показать переменные
clear        сбросить всё
help         справка
quit         выход
```

## Про типы

Главная идея проекта: глубина стека живёт в типе.

```idris
data Stack : Nat -> Type where
  Empty : Stack 0
  Push  : Double -> Stack n -> Stack (S n)
```

Из-за этого pop выглядит так:

```idris
pop : Stack (S n) -> (Double, Stack n)
```

Вызвать pop на пустом стеке нельзя физически, это ошибка компиляции.
То же самое с бинарными операциями: тип требует Stack (S (S n)),
то есть минимум два элемента, иначе не скомпилируется.

Проблема в том, что в REPL глубина стека неизвестна заранее, пользователь
кладёт и снимает что хочет. Решается через зависимую пару:

```idris
stack : (n : Nat ** Stack n)
```

Это просто пара из числа и стека такой глубины. Когда матчишь по конструкторам
стека, Idris сам понимает какой n получился в каждой ветке. Никакого ручного
отслеживания, никаких if n > 1.

## Структура

```
src/Stack.idr    типизированный стек, сюда смотреть в первую очередь
src/Env.idr      переменные, просто список пар
src/Parser.idr   разбор строки в тип Command
src/Main.idr     REPL, состояние передаётся через рекурсию без IORef
```
