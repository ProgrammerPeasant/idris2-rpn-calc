# rpn-calc — устройство проекта

RPN-калькулятор с переменными. Глубина стека отслеживается в типах на этапе
компиляции, поэтому операции над пустым стеком — ошибка компиляции, а не
падение в рантайме.

---

## Структура модулей

```
src/
  Stack.idr   — GADT стека; весь type-level интеллект здесь
  Env.idr     — переменные (список пар имя→значение)
  Parser.idr  — разбор текстовых команд в тип Command
  Main.idr    — REPL-цикл; держит состояние в обычной рекурсии
```

---

## Ключевое решение: Stack : Nat -> Type

```idris
data Stack : Nat -> Type where
  Empty : Stack 0
  Push  : Double -> Stack n -> Stack (S n)
```

`Stack` — это **зависимый тип (dependent type)**. Параметр `Nat` в типе — это
не значение времени выполнения, а **часть самого типа**. Idris2 не позволит
скомпилировать программу, если вы попытаетесь сделать pop на `Stack 0`:

```idris
pop : Stack (S n) -> (Double, Stack n)
```

Сигнатура `pop` требует `S n` — то есть хотя бы один элемент. Передать
`Stack 0` (пустой стек) нельзя: это ошибка **до запуска программы**. Никакого
`Maybe`, никаких исключений — просто компилятор говорит «нет».

То же с `binOp` (+, -, *, /):

```idris
binOp : (Double -> Double -> Double) -> Stack (S (S n)) -> Stack (S n)
```

Требует два элемента (`S (S n)`). Один элемент — не компилируется.

## Проблема REPL: экзистенциальная пара (DPair)

Глубина стека известна на этапе компиляции — хорошо для `pop` и `binOp`.
Но в REPL пользователь кладёт и снимает элементы в рантайме: глубина заранее
неизвестна.

Решение — **экзистенциальная пара**:

```idris
stack : (n : Nat ** Stack n)
```

Читается: «существует некоторое `n`, при котором это `Stack n`». Конкретное
значение `n` Idris выводит из паттерн-матчинга на конструкторах стека.

Поэтому состояние REPL выглядит так:

```idris
record ReplState where
  constructor MkReplState
  stack : (n : Nat ** Stack n)
  env   : Env
```

---

## Паттерн-матчинг уточняет тип

Когда нам нужна бинарная операция, мы не проверяем `n >= 2` через `if`.
Мы просто матчим по конструкторам GADT:

```idris
case s of
  Push y (Push x rest) =>
    -- здесь Idris знает: тип s был Stack (S (S k))
    -- x : Double, y : Double, rest : Stack k
    Right (MkReplState (_ ** Push (f x y) rest) env)
  _ =>
    Left "нужно минимум 2 элемента"
```

В ветке `Push y (Push x rest)` компилятор **сам выводит**, что глубина была
минимум 2. Результирующий стек `Push (f x y) rest` имеет тип `Stack (S k)` —
Idris выводит первый элемент DPair автоматически через `_`.

В ветке `_` Idris знает, что `n` — это 0 или 1, и не пытается обратиться к
несуществующим элементам.

## Конвенция RPN для бинарных операций

```idris
binOp f (Push y (Push x s)) = Push (f x y) s
```

`y` — вершина (последний push), `x` — под ней (первый push).
`f x y` означает: «то, что клали раньше» оператор «то, что клали позже».

Пример: `3 4 -` → push 3, push 4, `-` = f 3 4 = 3 − 4 = **−1**.
Это стандартная RPN-семантика (как в dc, HP-калькуляторах, Forth).

## Почему REPL без IORef

Состояние передаётся явно через рекурсию:

```idris
replLoop : ReplState -> IO ()
replLoop state = do
  ...
  replLoop state'   -- новое состояние — просто новый аргумент
```

Никакого изменяемого состояния. Это идиоматичный стиль функционального языка:
«изменение» состояния — это рекурсивный вызов с новым значением.

---

## Что происходит при некорректной команде

Вся логика команд живёт в чистой функции:

```idris
applyCommand : Command -> ReplState -> Either String ReplState
```

`Left err` — ошибка с сообщением, REPL печатает её и вызывает `replLoop`
со **старым** состоянием. `Right state'` — успех, стек обновляется.
IO-эффекты (печать) только в REPL-цикле, не в логике команд.

## Команды

| Ввод        | Действие                                      |
|-------------|-----------------------------------------------|
| `3.14`      | положить число на стек                        |
| `+ - * /`   | бинарная арифметика (нужно ≥ 2 элемента)      |
| `neg`       | сменить знак вершины (нужно ≥ 1)              |
| `dup`       | дублировать вершину                           |
| `drop`      | удалить вершину                               |
| `swap`      | поменять местами два верхних                  |
| `let имя`   | сохранить вершину в переменную                |
| `$имя`      | загрузить переменную на стек                  |
| `dump`      | показать текущий стек                         |
| `vars`      | показать все переменные                       |
| `clear`     | очистить всё                                  |
| `help`      | справка                                       |
| `quit`      | выход                                         |
